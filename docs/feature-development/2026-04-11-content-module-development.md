# 内容系统模块开发文档

> 基于 [社区项目MVP需求文档](../社区项目MVP需求文档.md) 和 [需求开发文档模板](../需求开发文档模板.md) 整理

## 1. 基本信息

- 需求名称：社区 MVP 内容系统模块（课程发布与阅读）
- 提出人：项目负责人（个人开发者）
- 负责人：个人开发者
- 预计完成时间：待排期
- 优先级：高
- 当前状态：切片01已完成（数据库与核心读写链路已落地）

## 2. 背景说明

- 业务背景：社区 MVP 的核心变现价值来自“会员可学习课程内容”，内容系统是会员权益兑现的关键模块。
- 问题描述：当前已完成用户模块，但课程相关能力尚未落地，导致会员激活后无法实际消费内容，业务闭环中断。
- 现有方案：总体 MVP 文档已定义 F5/F6/F7（课程发布、课程列表、课程详情与章节阅读）和 `course`、`course_chapter` 表结构，但尚未形成可直接执行的模块开发文档。
- 为什么现在要做：用户身份链路已具备，下一步应优先打通“管理员产出内容 -> 会员消费内容”的主路径，尽快验证付费社区核心价值。

## 3. 需求目标

- 主目标：实现内容系统第一阶段闭环，包括管理员发布课程、会员浏览课程列表、会员阅读章节内容。
- 次目标：为后续内容管理（编辑、下架、删除）、资源上传内嵌和讨论区导流提供稳定的内容基础。
- 成功标准：
  - 管理员可创建课程并维护章节，支持草稿与发布两种状态。
  - 非游客用户可查看已发布课程列表及课程简介。
  - 会员可查看课程详情及章节 Markdown 内容。
  - 章节按 `sort_order` 稳定展示，目录与正文渲染一致。

## 4. 非目标

- 不做内容：
  - 不实现课程购买流程（当前会员模式为 CDK 买断制）。
  - 不实现课程学习进度记录。
  - 不实现课程搜索、筛选和推荐。
  - 不实现章节评论与笔记能力。
  - 不实现复杂富文本编辑器，只要求支持 Markdown 文本输入。
- 暂不考虑的边界：
  - 大规模并发访问下的视频分发优化。
  - 多讲师、多作者协作发布。
  - 课程版本历史与回滚。
- 后续版本再做的内容：
  - 课程进度追踪（按用户记录章节阅读状态）。
  - 课程搜索与标签体系。
  - 内容统计看板（阅读量、完课率）。

## 5. 功能需求

- 需求 1：课程发布（管理员）
  - 管理员创建课程，填写标题、简介、封面图。
  - 管理员为课程添加多个章节，每章包含标题、Markdown 内容、排序号。
  - 支持课程状态保存为草稿或直接发布。
- 需求 2：课程列表
  - 课程列表仅对已登录会员开放（游客不可见）。
  - 列表展示封面图、课程标题、课程简介、章节数、发布时间。
  - 仅展示已发布课程，草稿课程不出现在公开列表。
- 需求 3：课程详情与章节阅读
  - 会员可查看课程章节目录与章节正文。
  - 章节正文按 Markdown 渲染，支持代码块、图片、视频 URL 嵌入。

如果按典型流程拆分，建议按以下步骤理解：

1. 管理员先在后台创建课程并添加章节，可选择草稿或发布。
2. 系统保存课程和章节信息，保证章节排序可稳定输出。
3. 会员访问课程列表，只能看到已发布课程。
4. 会员进入课程详情后查看章节目录与 Markdown 内容。
5. 未登录或无权限访问章节正文时，系统返回鉴权/鉴权失败提示。

## 6. 业务规则

- 课程与章节的创建、编辑、发布操作仅管理员可执行。
- 课程状态：`0=草稿`、`1=已发布`、`2=已下架`（下架用于后续管理能力）。
- 课程列表默认仅返回 `status=1` 的课程。
- 章节归属必须唯一绑定课程（`course_id` 必填且有效）。
- 同一课程内章节按 `sort_order` 升序展示；若排序号冲突，按创建时间升序兜底。
- 会员权限校验遵循用户状态：`status=1` 可阅读章节正文；其他状态不可阅读正文。
- Markdown 内容存储原文，渲染时统一走安全策略（至少防止明显脚本注入）。
- 视频内容以 URL 嵌入为主，MVP 阶段不做转码和 DRM。

## 7. 接口与数据

- 输入数据：
  - 课程创建：`title`、`description`、`coverImage`、`status`、`chapters[]`
  - 章节数据：`title`、`content`、`sortOrder`
  - 列表查询：分页参数（可选）
  - 详情查询：`courseId`
- 输出数据：
  - 课程创建结果：课程 ID、状态、章节数量
  - 课程列表结果：课程基础信息列表（不含完整章节正文）
  - 课程详情结果：课程基础信息、章节目录；会员场景返回章节正文
- 相关接口（建议）：
  - `POST /api/admin/courses`
  - `PUT /api/admin/courses/{id}`
  - `GET /api/courses`
  - `GET /api/courses/{id}`
  - `GET /api/courses/{id}/chapters/{chapterId}`
- 数据库表：
  - `course`
  - `course_chapter`
  - `user`（鉴权与会员状态判断）
- 消息事件：
  - MVP 阶段暂不引入消息总线事件
- 第三方依赖：
  - Markdown 渲染组件（前端）
  - OSS URL 访问能力（章节中的资源链接）

## 8. 验收标准

- 当管理员提交合法课程和章节数据时，应创建课程成功并可在后台再次查询到完整数据。
- 当非管理员调用课程创建或编辑接口时，应返回权限不足错误。
- 当课程状态为草稿时，普通课程列表接口不应返回该课程。
- 当会员访问课程列表时，应返回已发布课程并包含章节数量等基础字段。
- 当会员访问课程详情和章节内容时，应看到章节目录与 Markdown 正文。
- 当未登录或无权限用户访问章节正文接口时，应返回无权限错误且不泄露正文内容。
- 当章节排序号有效时，前端目录与后端返回顺序应一致。

建议按可测试表达补充：
- 当课程不存在时，详情接口应返回明确的资源不存在错误。
- 在章节为空的课程场景下，详情接口应返回空目录但不报错。
- 页面渲染应正确处理图片和视频 URL 的嵌入展示。

## 9. 边界与异常情况

- 边界情况 1：课程已发布但暂无章节，列表可见但详情目录为空。
- 边界情况 2：章节 `sort_order` 重复，系统需要定义稳定展示顺序。
- 异常情况 1：管理员提交章节内容过大（超字段长度或请求体限制）。
- 异常情况 2：章节中包含失效 OSS URL，渲染时出现资源加载失败。
- 异常情况 3：会员状态在会话中途变化（被禁用或降级），已持有 Token 访问章节应被拒绝。
- 异常情况 4：课程被下架后，旧链接访问应返回不可访问而不是历史缓存内容。

## 10. 约束条件

- 技术约束：
  - 后端采用 Spring Boot 3.x + MySQL 8.x。
  - 鉴权使用 JWT，并结合数据库用户状态做权限校验。
  - 内容格式采用 Markdown。
- 时间约束：
  - 本模块优先实现最小可用闭环，不引入搜索、统计等扩展能力。
- 兼容性约束：
  - 字段命名与状态定义需与总体 MVP 文档一致（`course.status`、用户 `status`）。
  - 接口返回结构应保持前后端契约稳定，避免频繁破坏性调整。
- 安全或合规约束：
  - Markdown 渲染需避免执行不可信脚本。
  - 未登录或无权限用户不可获取章节正文原文。
  - 管理员写操作接口必须做角色校验与审计日志预留点。
  - 内容治理采用“发布后处置 + 最小技术防护”策略，不引入全量预审。

## 11. 依赖与前置条件

- 前置需求：
  - 用户模块（注册、登录、鉴权、状态校验）已完成并可复用。
  - `course`、`course_chapter` 表已通过迁移脚本创建。
  - 基础异常处理、统一返回模型已具备。
- 外部依赖：
  - 前端 Markdown 渲染方案
  - OSS 文件访问 URL（用于课程封面与章节资源）
- 需要确认的信息：
  - 课程列表是否要求首版就支持分页与排序参数。
  - 章节正文是否通过课程详情一次性返回，还是拆分为独立章节详情接口。
  - 课程详情与章节正文是否均要求登录后访问（当前按“均要求登录”执行）。

## 12. 风险与待确认项

- 风险 1：当前社区为 CDK 注册制，非会员无法登录进入社区，该风险在当前版本范围内不成立。
- 风险 2：Markdown 渲染若未做安全处理，仍可能引入 XSS 风险（例如恶意脚本片段被拼接进页面执行）。
- 已确认 1：当前实现范围明确限定为内容系统（F5/F6/F7），暂不包含资源管理（F12/F13）与后台其他模块。
- 已确认 2：课程状态流转采用三态：草稿、已发布、已下架（下架状态用于后续管理能力）。

## 15. 简短示例

### 示例：课程章节访问控制

- 需求名称：课程章节访问控制
- 背景说明：会员权益需要通过章节内容访问权限体现。
- 需求目标：仅会员可读取章节正文，未登录或无权限用户不可读取正文。
- 非目标：不在本次实现学习进度记录。
- 功能需求：
  - 会员访问 `GET /api/courses/{id}/chapters/{chapterId}` 返回正文
  - 未登录或无权限访问同接口返回权限不足
  - 课程不存在返回资源不存在
- 验收标准：
  - 当用户为会员时，应返回章节 Markdown 正文
  - 当用户未登录或无权限时，不应返回正文字段
  - 当章节不属于课程时，应返回明确错误
- 风险与待确认项：
  - 课程详情接口是否需要预加载全部章节正文
  - 权限不足时前端提示文案是否统一复用
