## 使用方式

- 每完成一个较大的功能阶段，建议留一份单独记录。
- 记录内容建议包含：目标、实现范围、验证结果、遗留问题。
- 新窗口接手时，优先先看最近一条记录，再结合项目进度文档恢复上下文。
- 如果仓库已经有 `backend/`，也要先看 `backend/README.md` 和 `backend/pom.xml`，确认脚手架入口、环境要求和构建方式。
