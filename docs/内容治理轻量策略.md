# 内容治理轻量策略（单人维护版）

> 适用场景：CDK 注册制、小圈子程序员社区、单开发者维护。
> 目标：不做全量预审，在可控成本下把风险降到可接受水平，并保证可快速止损。

## 1. 治理原则

- 不做发布前全量人工审核，采用发布后处置。
- 不追求零风险，追求低成本可控风险。
- 风险治理采用“行为约束 + 最小技术防护 + 快速处置”三层。
- 只处理高风险和重复违规，不扩大到所有内容逐条人工检查。

## 2. 角色与责任

- 管理员（当前单人）：负责规则发布、违规判定、处置执行、记录留档。
- 社区成员：遵守内容规则，发现问题通过举报入口反馈。

## 3. 内容发布策略

- 默认允许会员发布与编辑（如后续开放帖子功能）。
- 默认不走预审，不设置发布等待队列。
- 命中明显高风险内容时可自动拦截或隐藏（见第 6 节）。

## 4. 违规分级与处置

### 4.1 轻度违规

- 示例：灌水、无意义重复、与主题明显无关。
- 处置：提醒或警告；必要时删除内容。

### 4.2 中度违规

- 示例：人身攻击、引战、恶意刷屏、重复发布广告。
- 处置：删除内容 + 短期禁言/限制发布；再次发生升级为重度。

### 4.3 重度违规

- 示例：恶意脚本投放、钓鱼链接、账号攻击、明显违法违规内容。
- 处置：立即下架内容 + 立即封禁账号；保留处理记录。

## 5. 处置时效目标（SLO）

- 高风险举报：24 小时内处理。
- 一般举报：72 小时内处理。
- 重度违规：发现后尽快处理，优先级最高。

## 6. 最小技术防护基线

- Markdown 渲染链路禁止执行不可信脚本。
- 对危险标签、危险属性和危险协议做白名单过滤或净化。
- 重要写操作保留最小审计字段：操作者、时间、对象、动作。
- 提供内容下架能力与账号封禁能力，支持快速止损。
- 允许用户举报，并在后台可查询待处理举报列表（后续迭代）。

## 7. 最小运营流程

1. 用户发布内容。
2. 系统执行基础安全过滤并保存。
3. 其他用户可举报。
4. 管理员查看举报并判定等级。
5. 执行处置：保留 / 删除 / 下架 / 封禁。
6. 记录处理结果，必要时向当事人给出简要说明。

## 8. 记录与复盘

- 每次重度违规建议记录一条复盘：触发点、处置动作、是否需要补技术防护。
- 同类问题连续出现 3 次以上，再考虑增加自动化规则或收紧权限。

## 9. 当前版本执行边界

- 当前内容模块范围仅包含课程发布与阅读（F5/F6/F7）。
- 本策略先作为统一治理口径，帖子/评论等社区交互模块上线时直接复用。
- 不在当前版本引入全量人工审核机制。
